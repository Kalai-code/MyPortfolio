Data Analysis on

ORCHID - Organ Retrieval and Collection of Health Information for Donation

Introduction:

Organ transplantation is a life-saving procedure for patients with advanced or end-stage diseases.This dataset can be used to identify opportunities to improve current practice, build tools to help OPOs make better decisions, and recommend policy changes to increase procurement. Downloaded the dataset from physionet for learning purposes.

Objective:

To increase transplant opportunities for patients by enhancing the procurement process, I followed a structured data analytics approach:

  • Data Understanding & Preparation – Examined the dataset to identify key variables, performed data cleaning to handle missing values and inconsistencies, and transformed the data for better usability.
  • Data Analysis & Processing – Loaded the cleaned data, conducted detailed analysis, and explored patterns in the procurement process.
  • Insights & Visual Storytelling – Generated meaningful insights and created visualizations to effectively increase the organ transplantation, highlighting key trends and correlations.

Solution:

Data Understanding

Tool Used: Jupyter Notebook

Thoroughly reviewed the information available on PhysioNet to understand the data collection process. Here are some key observations:

  • The dataset is organized around the typical process of organ procurement at an OPO. There are six key steps in the procurement process: referral, evaluation, approach, authorization, procurement, and transplant.
  • ORCHID consists of ten tables, each of which is provided as a comma separated value (CSV) file. Tables are linked via a unique patient identifier, PatientID. Each table falls into one of three categories: OPO Referrals, OPO Events, and OPO Deaths.
  • After gaining a clear understanding of the data, performed the next step in the Data Analysis Process: Data Cleaning.
